# Frontend changes — 22 Jun 2026

Batch of 14 fixes/features requested. Each item below lists **what** changed, **where**, and
whether it's fully done on the frontend or needs a backend follow-up (tracked in
`E:\tasleem\BACKEND_TODO_FINAL.md`).

---

### 1. Boosts no longer show negative  ✅
`/admin/stats` reports `boost_revenue` as a negative number. Boosts are income, so the dashboard
now uses `Math.abs()` for the **Revenue total** and the **Boosts** breakdown line.
- `src/views/admin/AdminDashboardView.vue` — `revenue` computed + `breakdown.boosts`.
- Backend should fix the sign at source (noted in BACKEND_TODO → "Minor").

### 2. Similar products on the product page now navigate  ✅
Clicking a similar product changed the route `:id` but the page didn't re-fetch (same component
reused). Extracted `loadProduct()` and added a `watch(route.params.id)` that resets per-product
state and reloads the product, images, reviews, AI similar/bundle, and sentiment.
- `src/views/products/ProductDetailView.vue`.

### 3. Cart pictures now appear  ✅
Cart rows fell back to `item.product?.image`, which the API doesn't expose (it uses `image_url`).
Now resolves through the shared `productImage()` helper.
- `src/views/CartView.vue` — `cartImg(item)` = `item.image || productImage(item.product)`.

### 4. Cart quantity stepper respects stock  ✅
The **+** button is disabled once quantity reaches available stock, with a "Max N in stock" hint;
attempting more shows a toast. (The cart store already clamps server-side as a backstop.)
- `src/views/CartView.vue` — `stockOf()`, `atMax()`, `incQty()`.

### 5. "Delivered within 3–5 business days" on order placed  ✅
Added a delivery-estimate banner to the checkout success step.
- `src/views/CheckoutView.vue` — step 4 success block.

### 6. Cancel only allowed while "placed" (pending)  ✅
Once an order is **confirmed** it can no longer be cancelled by the buyer (it's in progress).
Admin retains the ability to cancel/refund a confirmed order.
- `src/views/orders/OrdersView.vue` — `canCancel = s => s === 'pending'`.
- `src/views/orders/OrderDetailView.vue` — buyer `canCancel` is pending-only; `adminCanCancel`
  still covers pending+confirmed. Copy updated ("Cancellation is no longer available").

### 7. Edit price from My Listings (as a seller/user)  ✅
Inline price editing in the seller dashboard product table — click the price, type a new one, save.
Uses the JSON `productService.updateFields(id, { price })` (no full-form / image re-upload needed).
- `src/views/seller/SellerDashboardView.vue` — `startEdit()`, `savePrice()`, inline editor cell.

### 8. Checkout shows product type (Tasleem vs Marketplace)  ✅
Each line in the checkout order summary now carries a **Tasleem** or **Marketplace** badge.
- `src/views/CheckoutView.vue` — `isAdminOwned(item.product)` badge.

### 9. Orders bundling in My Orders  ✅
When one checkout mixes Tasleem and user products, My Orders now groups **all Tasleem items from
that checkout into one bundle card** (with the item list + combined total + "Cancel all"), while
**each user (C2C) order stays its own card**. Grouping keys Tasleem orders by creation minute
(same checkout); a lone Tasleem order renders as a normal single card.
- `src/views/orders/OrdersView.vue` — `groupedOrders`, `bundleTotal/Status/Cancellable`,
  `cancelBundle()`, and the bundle template.

### 10. Admin orders show the order ID  ✅
Added an **Order** column (`#id`, links to the order detail) to the admin orders table.
- `src/views/admin/AdminOrdersView.vue`.

### 11. Tasleem orders + delivery fees  🟡 backend
Tasleem orders **do** store `delivery_fee = 30`, but `/admin/stats → revenue.delivery_fees`
appears to exclude Tasleem orders, so Tasleem delivery doesn't show up as platform revenue.
Frontend already surfaces delivery in the "Platform fees" line + the new ledger; the aggregate
fix is backend-side (noted in BACKEND_TODO → "Tasleem delivery fees not counted").

### 12. Revenue breakdown — per-transaction detail (balance before + amount)  ✅
Added a **Transaction ledger** table to the admin dashboard: Type · Reference · Date ·
**Balance before** · **Amount** (signed) · **Balance after**, sourced from the account wallet
(`/wallet`). `balance_before` is derived as `balance_after − amount`.
- `src/views/admin/AdminDashboardView.vue` — ledger section + `txCredit/txBefore/txType/txRef`.
- (A dedicated platform-wide transactions endpoint would be richer than the account wallet, but
  none exists yet; this uses what the API provides.)

### 13. Everything builds & works  ✅
`npm run build` passes clean (8.8s, no errors). All touched views compile.
